Field Name           Type        Nullable    Description
-------------------------------------------------------------------------------
transaction_id      string      NO          Unique identifier for each transaction
                                            Format: TXN_YYYYMMDD_XXXXXXXX
                                            Example: TXN_20240115_001234567

customer_id         string      NO          Unique customer identifier
                                            Format: CUST_XXXXXXXX
                                            Example: CUST_00A1B2C3

                                            Format: ACC_XXXXXXXXXX
                                            Example: ACC_1234567890

                                            Format: YYYY-MM-DD HH:MM:SS
                                            Example: 2024-01-15 14:30:45

                                            Range: 0.01 - 1,000,000.00
                                            Precision: 2 decimal places

                                            Valid values: USD, EUR, GBP, JPY, CAD, AUD, CHF, CNY
                                            Example: USD

ip_address        string      YES         IPv4 or IPv6 address of transaction device
                                            Format: IPv4: xxx.xxx.xxx.xxx
                                            Example: 192.168.1.1

device_id         string      YES         Unique identifier for the device used
                                            Format: DEV_XXXXXXXXXX
                                            Example: DEV_A1B2C3D4E5

is_fraud          integer     NO          Fraud indicator (target variable)
                                            0: Legitimate transaction

7. LABEL DEFINITIONS
===============================================================================
Target Variable: is_fraud

-------------------------------------------------------------------------------
Label: 0 - Legitimate Transaction
-------------------------------------------------------------------------------
Definition: Transaction confirmed as legitimate with no fraudulent activity
Source: Customer confirmation, no chargeback, no dispute
Confidence: High (confirmed by customer or time-passed without dispute)

Examples:
- Regular purchase at grocery store by cardholder
- Bill payment from known account
- Transfer to known beneficiary

-------------------------------------------------------------------------------
Label: 1 - Confirmed Fraud
-------------------------------------------------------------------------------
Definition: Transaction confirmed as fraudulent through investigation
Source: Customer dispute, chargeback, internal investigation
Confidence: High (confirmed through investigation)

Sub-categories:
1. Card Not Present (CNP) Fraud: Online transaction without physical card
2. Card Present Fraud: Physical card stolen/cloned
3. Account Takeover: Fraudster gained access to account
4. Identity Fraud: Synthetic identity used
5. First-party Fraud: Customer claims fraud (friendly fraud)

Examples:
- Unauthorized transaction reported by customer
- Transaction after card reported stolen
- Transaction from unusual location with pattern matching known fraud

-------------------------------------------------------------------------------
Label Confidence Levels
-------------------------------------------------------------------------------
Confidence Level    Description
------------------ -------------------------------------------------------------
HIGH                Confirmed through investigation, chargeback, or customer report
MEDIUM              Suspicious pattern but not confirmed (e.g., velocity check)
LOW                 Algorithmic flag, pending review
-------------------------------------------------------------------------------

Fraud Label Sources
-------------------------------------------------------------------------------
Source Code        Description

DATA GOVERNANCE NOTES
===============================================================================

Data Retention Policy:
- Raw transactions: 7 years (regulatory requirement)
- Customer profiles: Indefinite (until account closure)
- Device fingerprints: 2 years
- Derived features: 90 days (for model training)
- Model predictions: 2 years (for audit trail)

Data Quality Requirements:
- Completeness: <5% missing values acceptable
- Accuracy: 99.9% accuracy required for critical fields
- Timeliness: Real-time for fraud detection (<100ms latency)
- Consistency: Cross-system validation required

Privacy & Compliance:
- PII fields: customer_id, name, email, phone (encrypted at rest)
- GDPR: Right to erasure implemented
- PCI DSS: Card data never stored in raw format
- Data masking applied for non-production environments

Update Frequency:
- Real-time: Transaction data, device fingerprints
- Hourly: Velocity features
- Daily: Customer profiles, merchant data
- Weekly: External enrichment data
- Monthly: Model retraining
