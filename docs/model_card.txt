1. MODEL OVERVIEW
================================================================================

1.1 Model Description
---------------------
The VeritasFinancial Fraud Detection System is a sophisticated ensemble model 
designed to detect fraudulent banking transactions in real-time. It combines:

- Classical ML (XGBoost): Fast, interpretable baseline predictions
- Deep Learning (Transformer): Sequential pattern recognition in transaction history
- Anomaly Detection (Isolation Forest): Unsupervised outlier detection
- LLM Reasoning Layer: Natural language explanations for fraud alerts

1.2 Intended Use
----------------
Primary Use Case: Real-time fraud detection for banking transactions
Secondary Use Cases: 
- Batch fraud analysis for compliance reporting
- Customer behavior profiling
- Risk scoring for new accounts
- Fraud pattern discovery

1.3 Target Users
----------------
- Fraud analysts reviewing flagged transactions
- Compliance officers monitoring suspicious activities
- Risk management teams analyzing fraud patterns
- Data scientists maintaining and improving the model

2.2 Key Innovations
-------------------
1. Grouped Query Attention (GQA): Reduces memory usage by 4× while maintaining performance
2. Rotary Position Embeddings (RoPE): Enables longer sequence handling (up to 500 txns)
3. Manifold-Constrained Hyper-Connections (mHC): Stabilizes training in deep networks
4. Adaptive Thresholding: Dynamically adjusts fraud threshold based on business rules

4.2 Feature Engineering Techniques
-----------------------------------
1. Log Transformations: Applied to skewed numerical features (amount, balance)
2. Standard Scaling: Z-score normalization for normally distributed features
3. Robust Scaling: Median/IQR scaling for features with outliers
4. One-Hot Encoding: For low-cardinality categorical features (<10 categories)
5. Target Encoding: For high-cardinality categorical features
6. Cyclical Encoding: For temporal features (hour_sin, hour_cos)
7. Rolling Statistics: Window-based aggregations (1h, 24h, 7d, 30d)
8. Difference Features: Current value vs historical average
9. Ratio Features: Amount ratios, time ratios
10. Interaction Features: Product of important features
11. Embedding Features: Learned embeddings for categorical variables
12. Graph Features: Network-based features from transaction graphs

9. MODEL LIMITATIONS
================================================================================

9.1 Known Limitations
---------------------
1. Cold Start Problem: New customers with no transaction history have lower accuracy
2. Concept Drift: Fraud patterns evolve; model needs monthly retraining
3. Adversarial Attacks: Sophisticated fraudsters may adapt to detection patterns
4. Data Quality: Relies on accurate device fingerprinting and IP geolocation
5. Regulatory Changes: Model must adapt to new compliance requirements
6. Cross-Border Complexity: Different fraud patterns in different countries
7. Real-Time Constraints: 10ms latency limit for real-time decisions
8. Interpretability Trade-off: Deep learning components less interpretable

9.2 Edge Cases
--------------
1. Very High Value Transactions (>$100k): Need manual review regardless of score
2. First Transaction: Use alternative scoring based on signup data
3. International Travelers: Higher false positives due to location changes
4. Shared Devices: Family members using same device may appear suspicious
5. Business Accounts: Different spending patterns than personal accounts
